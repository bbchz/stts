12  論理ベクトル

12.1 はじめに

この章では、論理ベクトル（logical vectors）を扱うためのツールについて学びます 。論理ベクトルは、各要素が TRUE（真）、FALSE（偽）、および NA（欠損値）という3つの可能な値のいずれかしか取ることができないため、最も単純な型のベクトルです 。生のデータの中に論理ベクトルが見つかることは比較的稀ですが、ほぼすべての分析の過程でそれらを作成し、操作することになります 。

まず、論理ベクトルを作成する最も一般的な方法である、数値の比較について説明します 。次に、ブール代数（Boolean algebra）を使用して異なる論理ベクトルを組み合わせる方法と、いくつかの有用な要約統計量について学びます 。最後に、論理ベクトルを活用して条件付きの変更を行うための2つの有用な関数、if_else() と case_when() で締めくくります 。

12.1.1 前提条件

この章で学ぶ関数のほとんどは base R（基本のR）で提供されているため、tidyverse は必須ではありませんが、データフレームを扱うために mutate() や filter() などを使用できるよう、やはりロードしておきます 。また、引き続き nycflights13::flights データセットから例を引用します 。

library(tidyverse)
library(nycflights13)
#> Warning: package 'nycflights13' was built under R version 4.5.3

しかし、より多くのツールをカバーし始めると、常に完璧な実例があるとは限りません 。そこで、c() を使ってダミーデータを作成することから始めます 。

x <- c(1, 2, 3, 5, 7, 11, 13)
x * 2
#> [1]  2  4  6 10 14 22 26

これにより、データの問題にどのように適用されるかを確認するのが難しくなる代わりに、個々の関数の説明が容易になります 。浮遊しているベクトル（free-floating vector）に対して行う操作は、mutate() などを使用してデータフレーム内の変数に対しても行えることを覚えておいてください 。

df <- tibble(x)
df |>
  mutate(y = x * 2)
#> # A tibble: 7 × 2
#>       x     y
#>   <dbl> <dbl>
#> 1     1     2
#> 2     2     4
#> 3     3     6
#> 4     5    10
#> 5     7    14
#> 6    11    22
#> # ℹ 1 more row

12.2 比較

論理ベクトルを作成する非常に一般的な方法は、<、<=、>、>=、!=、== を使用した数値の比較です 。これまでは、主に filter() 内で一時的に論理変数を作成してきました。それらは計算され、使用され、そして破棄されます 。たとえば、次のフィルタリングは、ほぼ時間通りに到着した日中の出発便をすべて見つけます 。

flights |>
  filter(dep_time > 600 & dep_time < 2000 & abs(arr_delay) < 20)
#> # A tibble: 172,286 × 19
#>    year month   day dep_time sched_dep_time dep_delay arr_time sched_arr_time
#>   <int> <int> <int>    <int>          <int>     <dbl>    <int>          <int>
#> 1  2013     1     1      601            600         1      844            850
#> 2  2013     1     1      602            610        -8      812            820
#> 3  2013     1     1      602            605        -3      821            805
#> 4  2013     1     1      606            610        -4      858            910
#> 5  2013     1     1      606            610        -4      837            845
#> 6  2013     1     1      607            607         0      858            915
#> # ℹ 172,280 more rows
#> # ℹ 11 more variables: arr_delay <dbl>, carrier <chr>, flight <int>, …

これがショートカットであることを知っておくと便利です。mutate() を使って、背後にある論理変数を明示的に作成することもできます 。

flights |>
  mutate(
    daytime = dep_time > 600 & dep_time < 2000,
    approx_ontime = abs(arr_delay) < 20,
    .keep = "used"
  )
#> # A tibble: 336,776 × 4
#>   dep_time arr_delay daytime approx_ontime
#>      <int>     <dbl> <lgl>   <lgl>        
#> 1      517        11 FALSE   TRUE         
#> 2      533        20 FALSE   FALSE        
#> 3      542        33 FALSE   FALSE        
#> 4      544       -18 FALSE   TRUE         
#> 5      554       -25 FALSE   FALSE        
#> 6      554        12 FALSE   TRUE         
#> # ℹ 336,770 more rows

これは、より複雑なロジックにおいて特に有用です。なぜなら、中間ステップに名前を付けることで、コードが読みやすくなり、各ステップが正しく計算されているかどうかの確認も容易になるからです 。

結局のところ、最初のフィルタリングは次と同等です 。

flights |>
  mutate(
    daytime = dep_time > 600 & dep_time < 2000,
    approx_ontime = abs(arr_delay) < 20,
  ) |>
  filter(daytime & approx_ontime)

12.2.1 浮動小数点の比較

数値に対して == を使用する際は注意してください 。たとえば、次のベクトルには数値の 1 と 2 が含まれているように見えます 。

x <- c(1 / 49 * 49, sqrt(2) ^ 2)
x
#> [1] 1 2

しかし、それらが等しいかどうかをテストすると、FALSE が返されます 。

x == c(1, 2)
#> [1] FALSE FALSE
何が起きているのでしょうか？ コンピュータは固定された小数点以下の桁数で数値を保存するため、1/49 や sqrt(2) を正確に表現する方法はなく、その後の計算はごくわずかにずれます 。digits 引数を指定して print() を呼び出すことで、正確な値を確認できます 。

R は通常、自動的に print を呼び出しますが（つまり x は print(x) のショートカットです）、他の引数を提供したい場合には明示的に呼び出すことが有用です 。

print(x, digits = 16)
#> [1] 0.9999999999999999 2.0000000000000004

R がデフォルトでこれらの数値を丸めて表示する理由がわかるでしょう 。それらは実際、期待される値に非常に近いのです 。

== が失敗する理由がわかったところで、どうすればよいでしょうか？ 一つの選択肢は、小さな差異を無視する dplyr::near() を使用することです 。

near(x, c(1, 2))
#> [1] TRUE TRUE

12.2.2 欠損値

欠損値（NA）は「不明（unknown）」を表すため、「伝染性」があります。不明な値を含むほぼすべての操作は、結果も不明になります 。

NA > 5
#> [1] NA
10 == NA
#> [1] NA

最も紛らわしい結果はこれです 。

NA == NA
#> [1] NA

これに少し文脈を与えてみると、なぜこれが真（TRUE）ではないのかを理解しやすくなります 。

# メアリーが何歳かわからない
age_mary <- NA

# ジョンが何歳かわからない
age_john <- NA

# メアリーとジョンは同い年か？
age_mary == age_john
#> [1] NA
# わかりません！

したがって、dep_time が欠損しているすべてのフライトを見つけたい場合、次のコードは機能しません。なぜなら、dep_time == NA はすべての行で NA を生成し、filter() は欠損値を自動的に除外するからです 。

flights |>
  filter(dep_time == NA)
#> # A tibble: 0 × 19
#> # ℹ 19 variables: year <int>, month <int>, day <int>, dep_time <int>,
#> #   sched_dep_time <int>, dep_delay <dbl>, arr_time <int>, …

代わりに、新しいツール is.na() が必要になります 。

12.2.3 is.na()

is.na(x) はあらゆる型のベクトルに対して機能し、欠損値に対しては TRUE を、それ以外に対しては FALSE を返します 。

is.na(c(TRUE, NA, FALSE))
#> [1] FALSE  TRUE FALSE
is.na(c(1, NA, 3))
#> [1] FALSE  TRUE FALSE
is.na(c("a", NA, "b"))
#> [1] FALSE  TRUE FALSE

is.na() を使用して、dep_time が欠損しているすべての行を見つけることができます 。

flights |>
  filter(is.na(dep_time))
#> # A tibble: 8,255 × 19
#>    year month   day dep_time sched_dep_time dep_delay arr_time sched_arr_time
#>   <int> <int> <int>    <int>          <int>     <dbl>    <int>          <int>
#> 1  2013     1     1       NA           1630        NA       NA           1815
#> 2  2013     1     1       NA           1935        NA       NA           2240
#> 3  2013     1     1       NA           1500        NA       NA           1825
#> 4  2013     1     1       NA            600        NA       NA            901
#> 5  2013     1     2       NA           1540        NA       NA           1747
#> 6  2013     1     2       NA           1620        NA       NA           1746
#> # ℹ 8,249 more rows
#> # ℹ 11 more variables: arr_delay <dbl>, carrier <chr>, flight <int>, …

12.3 ブール代数

複数の論理ベクトルが得られたら、ブール代数（Boolean algebra）を使用してそれらを組み合わせることができます 。R において、& は「かつ（and）」、| は「または（or）」、! は「ではない（not）」、そして xor() は「排他的論理和（exclusive or）」です 。たとえば、df |> filter(!is.na(x)) は x が欠損していないすべての行を見つけ、df |> filter(x < -10 | x > 0) は x が -10 未満または 0 より大きいすべての行を見つけます 。Figure 12.1 は、よく使われるブール演算の例とその仕組みを示しています 。

つまり、xor(x, y) は x が真、または y が真のいずれか一方のみである場合に真となります 。これは、私たちが日常の日本語で「AかBのどちらか」と言うときの意味に近いです 。「アイスクリームかケーキのどちらがいいですか？」という質問に対して、「両方」は通常、許容される答えではありません 。

【Geminiの補足】: 日常言語の「または」は、状況によって「どちらか一方（XOR）」を指す場合と「少なくとも一方（OR）」を指す場合がありますが、プログラミングにおける | は常に後者（両方の場合も含む）を意味します。一方のみを指したい場合に xor() を使用します。

R には & や | のほかに、&& や || もあります 。これらを dplyr の関数内で使用しないでください！ これらは「ショートサーキット（短絡評価）」演算子と呼ばれ、常に単一の TRUE または FALSE しか返しません 。これらは（制御構造などの）プログラミングにおいては重要ですが、データサイエンス（ベクトル操作）においては重要ではありません 。

12.3.1 欠損値

ブール代数における欠損値のルールは、一見すると一貫性がないように見えるため、説明が少し厄介です 。

df <- tibble(x = c(TRUE, FALSE, NA))

df |>
  mutate(
    and = x & NA,
    or = x | NA
  )
#> # A tibble: 3 × 3
#>   x     and   or   
#>   <lgl> <lgl> <lgl>
#> 1 TRUE  NA    TRUE 
#> 2 FALSE FALSE NA   
#> 3 NA    NA    NA

何が起きているかを理解するために、NA | TRUE（NA または TRUE）について考えてみましょう 。論理ベクトルにおける欠損値は、その値が TRUE または FALSE のいずれかになり得ることを意味します 。TRUE | TRUE も FALSE | TRUE も、少なくとも一方が TRUE であるため、どちらも TRUE です 。したがって、NA | TRUE も TRUE にならなければなりません。なぜなら NA が TRUE でも FALSE でも結果は変わらないからです 。しかし、NA | FALSE は、NA が TRUE か FALSE かによって結果が変わるため、NA になります 。

& に対しても、両方の条件が満たされなければならないことを考慮すると、同様の推論が当てはまります 。したがって、NA & TRUE は NA ですが（NA の値に依存するため）、NA & FALSE は少なくとも一方の条件が FALSE であることが確定しているため、FALSE になります 。

12.3.2 演算の順序

演算の順序は、自然言語（英語や日本語）のようには機能しないことに注意してください 。11月または12月に出発したすべてのフライトを見つける次のコードを見てみましょう 。

flights |>
   filter(month == 11 | month == 12)

これを、日常会話で「11月か12月に出発したフライトを見つけて」と言うように書きたくなるかもしれません 。

flights |>
   filter(month == 11 | 12)
#> # A tibble: 336,776 × 19
#>    year month   day dep_time sched_dep_time dep_delay arr_time sched_arr_time
#>   <int> <int> <int>    <int>          <int>     <dbl>    <int>          <int>
#> 1  2013     1     1      517            515         2      830            819
#> 2  2013     1     1      533            529         4      850            830
#> 3  2013     1     1      542            540         2      923            850
#> 4  2013     1     1      544            545        -1     1004           1022
#> 5  2013     1     1      554            600        -6      812            837
#> 6  2013     1     1      554            558        -4      740            728
#> # ℹ 336,770 more rows
#> # ℹ 11 more variables: arr_delay <dbl>, carrier <chr>, flight <int>, …

このコードはエラーになりませんが、期待通りには動作していないようです 。何が起きているのでしょうか？ ここで R はまず month == 11 を評価し、論理ベクトル（これを nov と呼びます）を作成します 。次に、nov | 12 を計算します 。論理演算子とともに数値を使用すると、0 以外のすべてが TRUE に変換されます。したがって、これは nov | TRUE と同等であり、常に TRUE になるため、すべての行が選択されてしまいます 。

flights |>
  mutate(
    nov = month == 11,
    final = nov | 12,
    .keep = "used"
  )
#> # A tibble: 336,776 × 3
#>   month nov   final
#>   <int> <lgl> <lgl>
#> 1     1 FALSE TRUE 
#> 2     1 FALSE TRUE 
#> 3     1 FALSE TRUE 
#> 4     1 FALSE TRUE 
#> 5     1 FALSE TRUE 
#> 6     1 FALSE TRUE 
#> # ℹ 336,770 more rows

12.3.3 %in%

== と | を正しい順序で記述する問題を避ける簡単な方法は、%in% を使用することです 。x %in% y は、x の値が y のどこかに含まれていれば TRUE となる、x と同じ長さの論理ベクトルを返します 。

1:12 %in% c(1, 5, 11)
#>  [1]  TRUE FALSE FALSE FALSE  TRUE FALSE FALSE FALSE FALSE FALSE  TRUE FALSE
letters[1:10] %in% c("a", "e", "i", "o", "u")
#>  [1]  TRUE FALSE FALSE FALSE  TRUE FALSE FALSE FALSE  TRUE FALSE

したがって、11月と12月のすべてのフライトを見つけるには、次のように書くことができます 。

flights |>
  filter(month %in% c(11, 12))

なお、%in% は NA に関して == とは異なるルールに従います。NA %in% NA は TRUE です 。

c(1, 2, NA) == NA
#> [1] NA NA NA
c(1, 2, NA) %in% NA
#> [1] FALSE FALSE  TRUE

これは便利なショートカットになります 。

flights |>
  filter(dep_time %in% c(NA, 0800))
#> # A tibble: 8,803 × 19
#>    year month   day dep_time sched_dep_time dep_delay arr_time sched_arr_time
#>   <int> <int> <int>    <int>          <int>     <dbl>    <int>          <int>
#> 1  2013     1     1      800            800         0     1022           1014
#> 2  2013     1     1      800            810       -10      949            955
#> 3  2013     1     1       NA           1630        NA       NA           1815
#> 4  2013     1     1       NA           1935        NA       NA           2240
#> 5  2013     1     1       NA           1500        NA       NA           1825
#> 6  2013     1     1       NA            600        NA       NA            901
#> # ℹ 8,797 more rows
#> # ℹ 11 more variables: arr_delay <dbl>, carrier <chr>, flight <int>, …

12.3.4 練習問題

  1. arr_delay は欠損しているが dep_delay は欠損していないすべてのフライトを見つけてください 。arr_time も sched_arr_time も欠損していないが、arr_delay が欠損しているすべてのフライトを見つけてください 。
  2. dep_time が欠損しているフライトは何件ありますか？ これらの行では他にどの変数が欠損していますか？ これらの行は何を表していると考えられますか？
  3. dep_time の欠損がフライトの欠航を意味すると仮定して、1日あたりの欠航便数を確認してください 。パターンはありますか？ 欠航便の割合と、欠航しなかった便の平均遅延時間との間に関連はありますか？

12.4 要約

以下のセクションでは、論理ベクトルを要約するためのいくつかの有用なテクニックを説明します 。論理ベクトル専用の関数だけでなく、数値ベクトルを扱うための関数も使用できます 。

12.4.1 論理値の要約

主な論理値の要約関数は any() と all() の2つです 。any(x) は |（または）に相当し、x の中に一つでも TRUE があれば TRUE を返します 。all(x) は &（かつ）に相当し、x のすべての値が TRUE である場合にのみ TRUE を返します 。ほとんどの要約関数と同様に、na.rm = TRUE を指定することで欠損値を除外できます 。

たとえば、all() と any() を使用して、すべてのフライトの出発遅延が最大でも1時間以内だったか、あるいは到着が5時間以上遅れたフライトがあったかどうかを確認できます 。group_by() を併用すれば、これを日ごとに調べることができます 。

flights |>
  group_by(year, month, day) |>
  summarize(
    all_delayed = all(dep_delay <= 60, na.rm = TRUE),
    any_long_delay = any(arr_delay >= 300, na.rm = TRUE),
    .groups = "drop"
  )
#> # A tibble: 365 × 5
#>    year month   day all_delayed any_long_delay
#>   <int> <int> <int> <lgl>       <lgl>         
#> 1  2013     1     1 FALSE       TRUE          
#> 2  2013     1     2 FALSE       TRUE          
#> 3  2013     1     3 FALSE       FALSE         
#> 4  2013     1     4 FALSE       FALSE         
#> 5  2013     1     5 FALSE       TRUE          
#> 6  2013     1     6 FALSE       FALSE         
#> # ℹ 359 more rows

しかし、ほとんどの場合、any() や all() は少し大雑把すぎます。いくつの値が TRUE または FALSE なのか、もう少し詳細を知りたいと思うでしょう 。そこで数値による要約の出番です 。

12.4.2 論理ベクトルの数値的要約

論理ベクトルを数値のコンテキストで使用すると、TRUE は 1 に、FALSE は 0 に変換されます 。これにより、sum() と mean() は論理ベクトルに対して非常に有用になります。sum(x) は TRUE の個数を与え、mean(x) は TRUE の割合を与えます（mean() は単に sum() を length() で割ったものだからです） 。

これにより、たとえば出発が最大でも1時間以内しか遅れなかったフライトの割合や、到着が5時間以上遅れたフライトの数を確認できます 。

flights |>
  group_by(year, month, day) |>
  summarize(
    proportion_delayed = mean(dep_delay <= 60, na.rm = TRUE),
    count_long_delay = sum(arr_delay >= 300, na.rm = TRUE),
    .groups = "drop"
  )
#> # A tibble: 365 × 5
#>    year month   day proportion_delayed count_long_delay
#>   <int> <int> <int>              <dbl>            <int>
#> 1  2013     1     1              0.939                3
#> 2  2013     1     2              0.914                3
#> 3  2013     1     3              0.941                0
#> 4  2013     1     4              0.953                0
#> 5  2013     1     5              0.964                1
#> 6  2013     1     6              0.959                0
#> # ℹ 359 more rows

12.4.3 論理値によるサブセッティング

要約における論理ベクトルの最後の用途は、論理ベクトルを使用して単一の変数を関心のあるサブセットにフィルタリングすることです 。これには base R の [（サブセット）演算子を使用します（詳細は Section 27.2 で学びます） 。

実際に遅延したフライトのみを対象に平均遅延時間を調べたいとしましょう 。一つの方法は、まずフライトをフィルタリングしてから平均を計算することです 。

flights |>
  filter(arr_delay > 0) |>
  group_by(year, month, day) |>
  summarize(
    behind = mean(arr_delay),
    n = n(),
    .groups = "drop"
  )
#> # A tibble: 365 × 5
#>    year month   day behind     n
#>   <int> <int> <int>  <dbl> <int>
#> 1  2013     1     1   32.5   461
#> 2  2013     1     2   32.0   535
#> 3  2013     1     3   27.7   460
#> 4  2013     1     4   28.3   297
#> 5  2013     1     5   22.6   238
#> 6  2013     1     6   24.4   381
#> # ℹ 359 more rows
これは機能しますが、予定より早く到着したフライトの平均遅延も計算したい場合はどうすればよいでしょうか？ 別のフィルタリングステップを実行し、その後2つのデータフレームを結合する方法を考え出す必要があります 。代わりに [ を使用してインラインでフィルタリングを行うことができます。arr_delay[arr_delay > 0] は、正の到着遅延のみを返します 。

これについては Chapter 19 で扱います 。

これにより、次のようになります。

flights |>
  group_by(year, month, day) |>
  summarize(
    behind = mean(arr_delay[arr_delay > 0], na.rm = TRUE),
    ahead = mean(arr_delay[arr_delay < 0], na.rm = TRUE),
    n = n(),
    .groups = "drop"
  )
#> # A tibble: 365 × 6
#>    year month   day behind ahead     n
#>   <int> <int> <int>  <dbl> <dbl> <int>
#> 1  2013     1     1   32.5 -12.5   842
#> 2  2013     1     2   32.0 -14.3   943
#> 3  2013     1     3   27.7 -18.2   914
#> 4  2013     1     4   28.3 -17.0   915
#> 5  2013     1     5   22.6 -14.0   720
#> 6  2013     1     6   24.4 -13.6   832
#> # ℹ 359 more rows

また、グループのサイズの違いにも注意してください。最初のコードでは n() は1日あたりの「遅延した」フライト数を与えますが、2番目のコードでは n() はフライトの「総数」を与えます 。

12.4.4 練習問題

  1. sum(is.na(x)) は何を教えてくれますか？ mean(is.na(x)) はどうでしょうか？
  2. 論理ベクトルに prod() を適用すると何を返しますか？ どの論理値要約関数と同等ですか？ 論理ベクトルに min() を適用すると何を返しますか？ どの論理値要約関数と同等ですか？ ドキュメントを読み、いくつか実験を行ってください 。

12.5 条件付き変換

論理ベクトルの最も強力な機能の一つは、条件付き変換への利用です。つまり、条件 x の場合には一つのことを行い、条件 y の場合には別のことを行うといった操作です 。これには、if_else() と case_when() という2つの重要なツールがあります 。

12.5.1 if_else()

ある条件が TRUE のときに一つの値を使い、FALSE のときに別の値を使いたい場合は、dplyr::if_else() を使用できます 。通常、if_else() の最初の3つの引数を使用します 。第1引数 condition は論理ベクトル、第2引数 true は条件が真の場合の出力、第3引数 false は条件が偽の場合の出力です 。

dplyr の if_else() は base R の ifelse() と非常によく似ています 。ifelse() に対する if_else() の主な利点は2つあります。欠損値に何が起きるかを選択できることと、変数の型が互換性のない場合に意味のあるエラーを出す可能性がはるかに高いことです 。

数値ベクトルを “+ve”（正）または “-ve”（負）としてラベル付けする簡単な例から始めましょう。

x <- c(-3:3, NA)
if_else(x > 0, "+ve", "-ve")
#> [1] "-ve" "-ve" "-ve" "-ve" "+ve" "+ve" "+ve" NA

オプションの第4引数 missing があり、入力が NA の場合に使用されます 。

if_else(x > 0, "+ve", "-ve", "???")
#> [1] "-ve" "-ve" "-ve" "-ve" "+ve" "+ve" "+ve" "???"

true および false 引数にベクトルを使用することもできます 。たとえば、これにより abs() の最小限の実装を作成できます 。

if_else(x < 0, -x, x)
#> [1]  3  2  1  0  1  2  3 NA

ここまではすべての引数で同じベクトルを使用してきましたが、もちろん組み合わせて使うこともできます 。たとえば、coalesce() の簡易版を次のように実装できます 。

x1 <- c(NA, 1, 2, NA)
y1 <- c(3, NA, 4, 6)
if_else(is.na(x1), y1, x1)
#> [1] 3 1 2 6

先ほどのラベル付けの例で、小さな不備に気づいたかもしれません。ゼロは正でも負でもありません 。さらにもう一つの if_else() を追加することで、これを解決できます 。

if_else(x == 0, "0", if_else(x < 0, "-ve", "+ve"), "???")
#> [1] "-ve" "-ve" "-ve" "0"   "+ve" "+ve" "+ve" "???"

これではすでに少し読みにくいですし、条件がさらに増えればもっと難しくなることは想像に難くないでしょう 。代わりに、dplyr::case_when() に切り替えることができます 。

12.5.2 case_when()

dplyr の case_when() は SQL の CASE 文に触発されたもので、異なる条件に対して異なる計算を行うための柔軟な方法を提供します 。これは、tidyverse で他に使うものとは似ていない特別な構文を持っています 。それは condition ~ output（条件 ~ 出力）というペアを取ります 。condition は論理ベクトルでなければならず、それが TRUE のときに output が使用されます 。

これにより、先ほどの入れ子になった if_else() を次のように再現できます 。

x <- c(-3:3, NA)
case_when(
  x == 0   ~ "0",
  x < 0    ~ "-ve",
  x > 0    ~ "+ve",
  is.na(x) ~ "???"
)
#> [1] "-ve" "-ve" "-ve" "0"   "+ve" "+ve" "+ve" "???"

コードの量は増えますが、より明示的でもあります 。case_when() の仕組みを説明するために、より単純なケースを見てみましょう 。どのケースにも一致しない場合、出力は NA になります 。

case_when(
  x < 0 ~ "-ve",
  x > 0 ~ "+ve"
)
#> [1] "-ve" "-ve" "-ve" NA    "+ve" "+ve" "+ve" NA

「デフォルト」や「その他すべて」の値を指定したい場合は、.default を使用します 。

case_when(
  x < 0 ~ "-ve",
  x > 0 ~ "+ve",
  .default = "???"
)
#> [1] "-ve" "-ve" "-ve" "???" "+ve" "+ve" "+ve" "???"

また、複数の条件が一致する場合、最初に一致したものだけが使用されることに注意してください 。

case_when(
  x > 0 ~ "+ve",
  x > 2 ~ "big"
)
#> [1] NA    NA    NA    NA    "+ve" "+ve" "+ve" NA

if_else() と同様に、~ の両側で変数を使用でき、問題に合わせて必要に応じて変数を組み合わせることができます 。たとえば、case_when() を使用して到着遅延に対して人間が読みやすいラベルを付けることができます 。

flights |>
  mutate(
    status = case_when(
      is.na(arr_delay)      ~ "cancelled",
      arr_delay < -30       ~ "very early",
      arr_delay < -15       ~ "early",
      abs(arr_delay) <= 15  ~ "on time",
      arr_delay < 60        ~ "late",
      arr_delay < Inf       ~ "very late",
    ),
    .keep = "used"
  )
#> # A tibble: 336,776 × 2
#>   arr_delay status 
#>       <dbl> <chr>  
#> 1        11 on time
#> 2        20 late   
#> 3        33 late   
#> 4       -18 early  
#> 5       -25 early  
#> 6        12 on time
#> # ℹ 336,770 more rows

このような複雑な case_when() 文を書くときは注意してください 。最初の数回の試行では < と > を混ぜて使ってしまい、誤って重複する条件を作ってしまいがちです 。

12.5.3 型の互換性

if_else() と case_when() はどちらも、出力において互換性（compatible）のある型を必要とすることに注意してください 。互換性がない場合、次のようなエラーが表示されます 。

if_else(TRUE, "a", 1)
#> Error in `if_else()`:
#> ! Can't combine `true` <character> and `false` <double>.

case_when(
  x < -1 ~ TRUE,
  x > 0  ~ now()
)
#> Error in `case_when()`:
#> ! Can't combine `..1 (right)` <logical> and `..2 (right)` <datetime<local>>.

ある型のベクトルを別の型に自動的に変換することはエラーの一般的な原因になるため、全体として互換性のある型は比較的少数です 。互換性のある主なケースは以下の通りです。

  • 数値ベクトルと論理ベクトルは互換性があります（Section 12.4.2 で議論した通りです） 。
  • 文字列と因子（Chapter 16）は互換性があります。因子は、値が制限された文字列と考えることができるからです 。
  • 日付と日時は互換性があります（Chapter 17 で議論します）。日付は日時の特殊なケースと考えることができるからです 。
  • NA（技術的には論理ベクトルです）は、すべてのベクトルが欠損値を表現する手段を持っているため、すべてと互換性があります 。

これらのルールを暗記する必要はありませんが、tidyverse 全体で一貫して適用されているため、時間が経てば自然に身につくでしょう 。

12.5.4 練習問題

  1. ある数値が偶数であるかどうかは、2で割り切れるかどうかでわかります。R では x %% 2 == 0 で確認できます 。この事実と if_else() を使用して、0 から 20 までの各数値が偶数か奇数かを判定してください 。
  2. x <- c("Monday", "Saturday", "Wednesday") のような曜日のベクトルが与えられたとき、if_else() 文を使用してそれらを週末（weekend）か平日（weekday）かでラベル付けしてください 。
  3. if_else() を使用して、数値ベクトル x の絶対値を計算してください 。
  4. flights の month 列と day 列を使用して、米国の重要な祝日（例：元日、独立記念日、感謝祭、クリスマス）をラベル付けする case_when() 文を書いてください 。まず祝日かどうかを示す論理列を作成し、次に祝日の名前を与えるか NA となる文字型の列を作成してください 。

12.6 まとめ

論理ベクトルの定義は、各値が TRUE、FALSE、または NA のいずれかでなければならないため、単純です 。しかし、論理ベクトルは非常に大きなパワーを提供します。この章では、>、<、<=、>=、==、!=、および is.na() を使用して論理ベクトルを作成する方法、それらを !、&、| で組み合わせる方法、そして any()、all()、sum()、mean() で要約する方法を学びました 。また、論理ベクトルの値に応じて値を返すことができる強力な if_else() および case_when() 関数についても学びました 。

論理ベクトルは、以降の章でも繰り返し登場します 。たとえば Chapter 14 では、パターンに一致する要素に対して TRUE を返す str_detect(x, pattern) について学びます。また Chapter 17 では、日付や時刻の比較から論理ベクトルを作成します 。しかし、今のところは次に重要な型のベクトルである数値ベクトル（numeric vectors）に進みましょう 。
