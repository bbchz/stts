24  ウェブスクレイピング

24.1 はじめに

(https://rvest.tidyverse.org) を使ったウェブスクレイピングの基本を紹介します 。ウェブスクレイピングは、ウェブページからデータを抽出するための非常に便利なツールです 。一部のウェブサイトでは、API（構造化された HTTP リクエストのセットで、JSON としてデータを返すもの）を提供しており、これについては Chapter 23 のテクニックを用いて処理します 。API が利用可能な場合は、通常より信頼性の高いデータが得られるため、そちらを使用すべきです 。残念ながら、ウェブ API を使ったプログラミングはこの本の範囲外です 。代わりに、この章ではサイトが API を提供しているかどうかにかかわらず機能するテクニックである、スクレイピングについて教えます 。

この章では、HTML の基本に飛び込む前に、まずスクレイピングの倫理性と合法性について議論します 。次に、ページ上の特定の要素を特定するための CSS セレクタの基本と、rvest の関数を使って HTML からテキストや属性を取り出し R に取り込む方法を学びます 。その後、スクレイピングしたいページに必要な CSS セレクタを見つけるためのいくつかのテクニックを検討し、最後にいくつかのケーススタディと動的なウェブサイトに関する短い議論で締めくくります 。

24.1.1 事前準備

この章では、rvest パッケージが提供するツールに焦点を当てます 。rvest は tidyverse のメンバーですが、コアメンバーではないため、明示的にロードする必要があります 。また、スクレイピングしたデータを扱う際に全般的に役立つため、tidyverse もロードします 。

library(tidyverse)
library(rvest)

24.2 スクレイピングの倫理性と合法性

ウェブスクレイピングを実行するために必要なコードについて話し始める前に、それを行うことが法的および倫理的に問題ないかどうかについて話す必要があります 。全体として、これら両方に関する状況は複雑です。

合法性は、あなたがどこに住んでいるかに大きく依存します 。しかし、一般的な原則として、データが公開されており、個人的なものではなく、事実に基づいたものであれば、おそらく問題ないでしょう 。これら3つの要因は、以下で説明するように、サイトの利用規約、個人を特定できる情報、および著作権に関連しているため重要です 。もしデータが公開されていない、個人的なものである、事実に基づいたものではない場合、あるいは特にそのデータを使ってお金を稼ぐためにスクレイピングを行う場合は、弁護士に相談する必要があります 。いかなる場合でも、スクレイピングしているページをホストしているサーバーのリソースを尊重すべきです 。最も重要なことは、多くのページをスクレイピングする場合、各リクエストの間に少し時間を置くようにすることです 。その簡単な方法の一つは、Dmytro Perepolkin による polite パッケージを使用することです 。これはリクエスト間に自動的に一時停止を入れ、結果をキャッシュするため、同じページを二度要求することはありません 。

24.2.1 利用規約

注意深く見ると、多くのウェブサイトのページのどこかに「利用規約（terms and conditions）」や「サービス利用規約（terms of service）」へのリンクが含まれていることに気づくでしょう。そのページをよく読むと、サイトがウェブスクレイピングを明確に禁止していることがよくあります 。これらのページは、企業が非常に広範な主張を行う法的な先占（land grab）の場となる傾向があります 。可能な限りこれらの利用規約を尊重するのが礼儀ですが、いかなる主張も鵜呑みにはしないでください 。米国の裁判所は、単にウェブサイトのフッターに利用規約を記載するだけでは、ユーザーがそれに拘束されるには不十分であるとの判断を一般的に下しています（例：HiQ Labs v. LinkedIn）。一般的に、利用規約に拘束されるためには、アカウントの作成やチェックボックスへのチェックなど、何らかの明示的な行動をとっている必要があります 。これが、データが公開（public）されているかどうかが重要である理由です 。データにアクセスするためにアカウントが必要ない場合、利用規約に拘束される可能性は低いです 。ただし、欧州では状況がかなり異なり、裁判所は明示的に同意しなくても利用規約に法的強制力があるとの判断を下していることに注意してください 。

24.2.2 個人を特定できる情報

たとえデータが公開されていても、名前、メールアドレス、電話番号、生年月日などの個人を特定できる情報（PII）のスクレイピングには細心の注意を払う必要があります。欧州にはこのようなデータの収集や保存に関する特に厳格な法律（GDPR）があり、あなたがどこに住んでいようとも、倫理的な泥沼に足を踏み入れる可能性があります 。たとえば2016年に、ある研究者グループがデートサイト OkCupid 上の約7万人に関する公開プロフィール情報（ユーザー名、年齢、性別、場所など）をスクレイピングし、匿名化の試みを一切行わずにこれらのデータを公開しました 。研究者たちは、データがすでに公開されていたため問題はないと考えていましたが、この研究はデータセット内で公開されたユーザーの特定可能性をめぐる倫理的な懸念から広く非難されました 。もしあなたの仕事に個人を特定できる情報のスクレイピングが含まれる場合は、OkCupid の研究 や、個人情報の取得と公開を伴う同様の倫理的に疑問のある研究について読むことを強くお勧めします。

24.2.3 著作権

最後に、著作権法についても考慮する必要があります 。著作権法は複雑ですが、何が保護されるかを正確に記述している 米国法 を見てみる価値があります。「[…] 表現の有形の媒体に固定された著作者の独創的な著作物 […]」とあります 。続いて、文学作品、音楽作品、映画などの適用される特定のカテゴリーが記述されています 。注目すべきことに、著作権保護から欠けているのはデータです。これは、事実（fact）のみをスクレイピングの対象とする限り、著作権保護は適用されないことを意味します 。（ただし、欧州にはデータベースを保護する独立した「sui generis（特有の）」権利があることに注意してください。）

簡単な例として、米国では材料のリストや手順は著作権の対象にならないため、著作権を使ってレシピを保護することはできません 。しかし、そのレシピのリストにかなりの量の独創的な文学的内容が伴っている場合、それは著作権の対象となります 。インターネットでレシピを探すと、決まってその前に非常に多くのコンテンツがあるのはこのためです 。もし独創的なコンテンツ（テキストや画像など）をスクレイピングする必要がある場合でも、フェアユースの法理（doctrine of fair use） の下で保護される可能性があります 。フェアユースは一律の規則ではなく、多くの要因を秤にかけます 。研究や非営利目的でデータを収集しており、スクレイピングするものを必要な範囲に限定していれば、適用される可能性が高くなります 。

24.3 HTML の基本

ウェブページをスクレイピングするには、まずウェブページを記述する言語である HTML について少し理解する必要があります 。HTML は HyperText Markup Language の略で、次のようになります 。

<html>
<head>
  <title>ページのタイトル</title>
</head>
<body>
  <h1 id='first'>見出し</h1>
  <p>いくつかのテキスト &amp; <b>太字のテキスト。</b></p>
  <img src='myimg.png' width='100' height='100'>
</body>

HTML は、開始タグ（例：<tag>）、オプションの属性（attributes）（id='first'）、終了タグ（</tag> など）、および内容（contents）（開始タグと終了タグの間のすべて）で構成される要素（elements）によって形成される階層構造を持っています 。< と > は開始タグと終了タグに使用されるため、直接書くことはできません 。代わりに、HTML のエスケープ（escapes）である &gt;（より大きい）と &lt;（より小さい）を使用する必要があります 。そして、これらのエスケープは & を使用するため、文字通りのアンパサンドが必要な場合は &amp; としてエスケープしなければなりません 。多種多様な HTML エスケープが存在しますが、rvest が自動的に処理してくれるため、あまり心配する必要はありません 。ウェブスクレイピングが可能なのは、スクレイピングしたいデータを含むほとんどのページが、一般的に一貫した構造を持っているからです 。

24.3.1 要素

HTML 要素は 100 種類以上あります。その中でも特に重要なものをいくつか挙げます。

  • すべての HTML ページは <html> 要素の中にあり、2 つの子要素を持つ必要があります。それは、ページのタイトルのようなドキュメントのメタデータを含む <head> と、ブラウザで見えるコンテンツを含む <body> です 。
  • <h1>（見出し1）、<section>（セクション）、<p>（段落）、<ol>（順序付きリスト）などのブロックタグは、ページの全体的な構造を形成します 。
  • <b>（太字）、<i>（斜体）、<a>（リンク）などのインラインタグは、ブロックタグ内のテキストをフォーマットします 。

見たことのないタグに遭遇した場合は、少しググればその機能を調べることができます 。また、ウェブプログラミングのほぼすべての側面を記述している MDN Web Docs も良い出発点です 。

ほとんどの要素は、開始タグと終了タグの間に内容を持つことができます 。この内容はテキストである場合もあれば、さらなる要素である場合もあります。たとえば、次の HTML には、1 つの単語が太字になったテキストの段落が含まれています 。

<p>
  こんにちは！私の<b>名前</b>はハドリーです。
</p>

子（children）とはそこに含まれる要素のことで、上の <p> 要素には 1 つの子要素である <b> 要素があります 。<b> 要素には子要素はありませんが、内容（テキストの「名前」）を持っています 。

24.3.2 属性

タグは、name1='value1' name2='value2' のような形式の、名前付きの属性（attributes）を持つことができます 。最も重要な属性の 2 つは id と class で、これらは CSS（Cascading Style Sheets）と組み合わせてページの視覚的な外見を制御するために使用されます 。これらはページからデータをスクレイピングする際にしばしば役立ちます 。また、属性はリンクの遷移先（<a> 要素の href 属性）や画像のソース（<img> 要素の src 属性）を記録するためにも使用されます 。

24.4 データの抽出

スクレイピングを開始するには、スクレイピングしたいページの URL が必要です。通常はウェブブラウザからコピーできます 。次に、read_html() を使ってそのページの HTML を R に読み込む必要があります 。これは xml_document オブジェクトを返し、それを rvest の関数を使って操作します 。

html <- read_html("http://rvest.tidyverse.org/")
html
#> {html_document}
#> <html lang="en">
#> [1] <head>\n<meta http-equiv="Content-Type" content="text/html; charset=UT ...
#> [2] <body>\n    <a href="#container" class="visually-hidden-focusable">Ski ...

rvest には、HTML をインラインで書くことができる関数も含まれています 。この章では、さまざまな rvest 関数がどのように機能するかを簡単な例で教えるために、これを多用します 。

html <- minimal_html("
  <p>これは段落です</p>
  <ul>
    <li>これは箇条書きリストです</li>
  </ul>
")
html
#> {html_document}
#> <html>
#> [1] <head>\n<meta http-equiv="Content-Type" content="text/html; charset=UT ...
#> [2] <body>\n<p>これは段落です</p>\n  <ul>\n<li>これは箇条書きリストです</li>\n  </ul>\n</body>

R に HTML を取り込んだら、次は関心のあるデータを抽出します 。まず、関心のある要素を特定するための CSS セレクタと、それらからデータを抽出するために使用できる rvest 関数について学びます 。次に、特別なツールがある HTML テーブルについて簡単に説明します 。

24.4.1 要素を見つける

CSS は Cascading Style Sheets の略で、HTML ドキュメントの視覚的なスタイリングを定義するためのツールです 。CSS には、CSS セレクタと呼ばれる、ページ上の要素を選択するためのミニ言語が含まれています 。CSS セレクタは HTML 要素を特定するためのパターンを定義するもので、抽出したい要素を簡潔に記述する方法を提供するため、スクレイピングに役立ちます 。CSS セレクタについては Section 24.5 で詳しく説明しますが、幸いなことに、次の 3 つだけでかなりのことができます 。

  • p は、すべての <p> 要素を選択します 。
  • .title は、class が “title” であるすべての要素を選択します 。
  • #title は、id 属性が “title” に等しい要素を選択します 。ID 属性はドキュメント内で一意である必要があるため、これは常に単一の要素のみを選択します 。

これらのセレクタを簡単な例で試してみましょう。

html <- minimal_html("
  <h1>これは見出しです</h1>
  <p id='first'>これは段落です</p>
  <p class='important'>これは重要な段落です</p>
")

html_elements() を使って、セレクタに一致するすべての要素を見つけます。

html |> html_elements("p")
#> {xml_nodeset (2)}
#> [1] <p id="first">これは段落です</p>
#> [2] <p class="important">これは重要な段落です</p>
html |> html_elements(".important")
#> {xml_nodeset (1)}
#> [1] <p class="important">これは重要な段落です</p>
html |> html_elements("#first")
#> {xml_nodeset (1)}
#> [1] <p id="first">これは段落です</p>

もう一つの重要な関数は html_element() で、これは常に入力と同じ数の出力を返します 。ドキュメント全体に適用すると、最初のマッチを返します。

html |> html_element("p")
#> {html_node}
#> <p id="first">

要素に一致しないセレクタを使用した場合の html_element() と html_elements() の間には重要な違いがあります 。html_elements() は長さ 0 のベクトルを返しますが、html_element() は欠損値を返します 。

html |> html_elements("b")
#> {xml_nodeset (0)}
html |> html_element("b")
#> {xml_missing}
#> <NA>

24.4.2 選択の入れ子

ほとんどの場合、html_elements() と html_element() を組み合わせて使用します。通常は、html_elements() を使って「観測値（observations）」となる要素を特定し、次に html_element() を使って「変数（variables）」となる要素を見つけます 。簡単な例でこれを確認してみましょう。ここに、スター・ウォーズの 4 人のキャラクターに関する情報を各リスト項目（<li>）に含む順序なしリスト（<ul>）があります。

html <- minimal_html("
  <ul>
    <li><b>C-3PO</b> は、体重 <span class='weight'>167 kg</span> の <i>ドロイド</i> です</li>
    <li><b>R4-P17</b> は <i>ドロイド</i> です</li>
    <li><b>R2-D2</b> は、体重 <span class='weight'>96 kg</span> の <i>ドロイド</i> です</li>
    <li><b>ヨーダ</b> の体重は <span class='weight'>66 kg</span> です</li>
  </ul>
  ")

html_elements() を使って、各要素が異なるキャラクターに対応するベクトルを作成できます。

characters <- html |> html_elements("li")
characters
#> {xml_nodeset (4)}
#> [1] <li>\n<b>C-3PO</b> は、体重 <span class="weight">167 kg</span> の <i>ドロイド</ ...
#> [2] <li>\n<b>R4-P17</b> は <i>ドロイド</i> です</li>
#> [3] <li>\n<b>R2-D2</b> は、体重 <span class="weight">96 kg</span> の <i>ドロイド</i ...
#> [4] <li>\n<b>ヨーダ</b> の体重は <span class="weight">66 kg</span> です</li>

各キャラクターの名前を抽出するには、html_element() を使用します。なぜなら、html_elements() の出力に適用した場合、要素ごとに 1 つのレスポンスを返すことが保証されているからです。

characters |> html_element("b")
#> {xml_nodeset (4)}
#> [1] <b>C-3PO</b>
#> [2] <b>R4-P17</b>
#> [3] <b>R2-D2</b>
#> [4] <b>ヨーダ</b>

名前に関しては html_element() と html_elements() の区別は重要ではありませんが、体重（weight）に関しては重要です 。体重の <span> がなくても、各キャラクターに対して 1 つの体重を取得したいからです 。それが html_element() の役割です。

characters |> html_element(".weight")
#> {xml_nodeset (4)}
#> [1] <span class="weight">167 kg</span>
#> [2] NA
#> [3] <span class="weight">96 kg</span>
#> [4] <span class="weight">66 kg</span>

html_elements() は、characters の子であるすべての体重 <span> を見つけます 。これらは 3 つしかないため、名前と体重の対応関係が失われてしまいます。

characters |> html_elements(".weight")
#> {xml_nodeset (3)}
#> [1] <span class="weight">167 kg</span>
#> [2] <span class="weight">96 kg</span>
#> [3] <span class="weight">66 kg</span>

24.4.3 テキストと属性

html_text2() は、HTML 要素のプレーンテキストの内容を抽出します 。

characters |>
  html_element("b") |>
  html_text2()
#> [1] "C-3PO"  "R4-P17" "R2-D2"  "ヨーダ"

characters |>
  html_element(".weight") |>
  html_text2()
#> [1] "167 kg" NA       "96 kg"  "66 kg"

エスケープは自動的に処理されることに注意してください 。ソース HTML 内では HTML エスケープが見られますが、rvest から返されるデータ内では見られません 。

html_attr() は属性からデータを抽出します。

html <- minimal_html("
  <p><a href='https://en.wikipedia.org/wiki/Cat'>猫</a></p>
  <p><a href='https://en.wikipedia.org/wiki/Dog'>犬</a></p>
")

html |>
  html_elements("p") |>
  html_element("a") |>
  html_attr("href")
#> [1] "https://en.wikipedia.org/wiki/Cat" "https://en.wikipedia.org/wiki/Dog"

html_attr() は常に文字列を返すため、数値や日付を抽出している場合は、後処理が必要になります 。

24.4.4 テーブル

運が良ければ、データはすでに HTML テーブルに保存されており、そのテーブルから読み取るだけという場合もあります 。ブラウザでテーブルを認識するのは通常簡単です。行と列の長方形の構造を持っており、Excel のようなツールにコピー＆ペーストできます 。HTML テーブルは、主に 4 つの要素、<table>、<tr>（テーブルの行）、<th>（テーブルの見出し）、および <td>（テーブルのデータ）から構築されます 。以下は、2 列 3 行の単純な HTML テーブルです。

html <- minimal_html("
  <table class='mytable'>
    <tr><th>x</th>   <th>y</th></tr>
    <tr><td>1.5</td> <td>2.7</td></tr>
    <tr><td>4.9</td> <td>1.3</td></tr>
    <tr><td>7.2</td> <td>8.1</td></tr>
  </table>
  ")

rvest は、この種のデータを読み取る方法を知っている関数 html_table() を提供しています 。これは、ページ上で見つかったテーブルごとに 1 つの tibble を含むリストを返します 。html_element() を使って、抽出したいテーブルを特定してください。

html |>
  html_element(".mytable") |>
  html_table()
#> # A tibble: 3 × 2
#>       x     y
#>   <dbl> <dbl>
#> 1   1.5   2.7
#> 2   4.9   1.3
#> 3   7.2   8.1

x と y が自動的に数値に変換されていることに注意してください 。この自動変換は常に機能するとは限らないため、より複雑なシナリオでは convert = FALSE でオフにし、独自の変換を行うことができます 。

24.5 適切なセレクタを見つける

データに必要なセレクタを見極めることが、通常、問題の最も困難な部分です 。特定の（つまり、関心のないものを選択しない）かつ感度の高い（つまり、関心のあるものをすべて選択する）セレクタを見つけるために、しばしば実験を行う必要があります 。多くの試行錯誤がこのプロセスの通常の姿です 。

このプロセスを支援するために利用可能な主なツールが 2 つあります。SelectorGadget と、ブラウザの開発者ツールです 。

SelectorGadget は、提供された肯定的な例と否定的な例に基づいて CSS セレクタを自動的に生成する JavaScript のブックマークレットです 。常に機能するとは限りませんが、機能するときは魔法のようです 。

すべてのモダンなブラウザには開発者向けのツールキットが付属していますが、普段使っているブラウザでなくても Chrome をお勧めします。ウェブ開発者ツールが非常に優れており、すぐに利用できるからです 。

ページ上の要素を右クリックして 検証（Inspect） をクリックします 。すると、クリックした要素を中心とした、HTML ページ全体の展開可能なビューが開きます 。これを使ってページを探索し、どのようなセレクタが機能しそうかの感覚をつかむことができます 。特に class 属性と id 属性に注目してください。これらはページの視覚的な構造を形成するためにしばしば使用されるため、探しているデータを抽出するための優れたツールになります 。

要素ビューの中で要素を右クリックし、セレクタをコピー（Copy as Selector） を選択すると、関心のある要素を一意に特定するセレクタを生成することもできます 。

もし SelectorGadget や Chrome の開発者ツールが生成した CSS セレクタが理解できない場合は、CSS セレクタを分かりやすい英語に翻訳してくれる Selectors Explained を試してみてください 。これを頻繁に行うようになるなら、CSS セレクタ全般についてもっと学びたいと思うかもしれません 。楽しい CSS dinner チュートリアルから始めて、MDN web docs を参照することをお勧めします 。

24.6 すべてをまとめる

これらすべてをまとめて、いくつかのウェブサイトをスクレイピングしてみましょう 。これらの例を実行したときに、すでに機能しなくなっているリスクがあります。それがウェブスクレイピングの根本的な課題です 。サイトの構造が変われば、スクレイピングのコードも変えなければなりません 。

24.6.1 スター・ウォーズ

rvest には vignette("starwars") に非常に簡単な例が含まれています。これは最小限の HTML を持つシンプルなページなので、始めるのに適した場所です 。今すぐそのページに移動し、「要素を検証」して、スター・ウォーズ映画のタイトルである見出しの一つを調べてみることをお勧めします 。キーボードやマウスを使って HTML の階層を探索し、各映画で使用されている共通の構造を感じ取れるか試してみてください 。

各映画が次のような共通の構造を持っていることがわかるはずです。

<section>
  <h2 data-id="1">ファントム・メナス</h2>
  <p>公開日: 1999-05-19</p>
  <p>監督: <span class="director">ジョージ・ルーカス</span></p>

  <div class="crawl">
    <p>...</p>
    <p>...</p>
    <p>...</p>
  </div>
</section>

私たちの目標は、このデータを title（タイトル）、year（年）、director（監督）、intro（イントロ）という変数を持つ 7 行のデータフレームにすることです 。まず HTML を読み込み、すべての <section> 要素を抽出することから始めます。

url <- "https://rvest.tidyverse.org/articles/starwars.html"
html <- read_html(url)

section <- html |> html_elements("section")

section |> html_element("h2") |> html_text2()

24.6.2 IMDb のトップ映画

次のタスクとして、もう少し難しいもの、インターネット映画データベース（IMDb）からトップ 250 の映画を抽出することに挑戦します 。この章を書いた時点では、ページは Figure 24.1 のようになっていました。

このデータには明確な表構造があるため、html_table() から始める価値があります 。

url <- "https://web.archive.org/web/20220201012049/https://www.imdb.com/chart/top/"
html <- read_html(url)

table <- html |>
  html_element("table") |>
  html_table()

これにはいくつかの空の列が含まれていますが、全体としてテーブルからの情報をうまく捉えています 。しかし、使いやすくするためにはさらなる処理が必要です 。まず、列名を扱いやすいものに変更し、順位（rank）とタイトル（title）にある余分な空白を削除します 。select() を使って、これら 2 つの列の選択と名前変更を 1 ステップで行います 。次に、改行と余分なスペースを削除し、Section 15.3.4 で学んだ separate_wider_regex() を適用して、タイトル、年、順位をそれぞれの変数に引き出します 。

ratings <- table |>
  select(
    rank_title_year = `Rank & Title`,
    rating = `IMDb Rating`
  ) |>
  mutate(
    rank_title_year = str_replace_all(rank_title_year, "\n +", " ")
  ) |>
  separate_wider_regex(
    rank_title_year,
    patterns = c(
      rank = "\\d+", "\\. ",
      title = ".+", " +\\(",
      year = "\\d+", "\\)"
    )
  )

ほとんどのデータがテーブルのセルから得られるこのような場合でも、生の HTML を見てみる価値はあります 。そうすれば、属性の一つを使用することで、少し余分なデータを追加できることがわかるでしょう 。これが、ページのソースを探索する（spelunking）ことに少し時間を費やす価値がある理由の一つです。余分なデータが見つかるかもしれませんし、少し簡単なパース経路が見つかるかもしれません 。

html |>
  html_elements("td strong") |>
  head() |>
  html_attr("title")
#> [1] "9.2 based on 2,536,415 user ratings"
#> [2] "9.1 based on 1,745,675 user ratings"
#> [3] "9.0 based on 1,211,032 user ratings"
#> [4] "9.0 based on 2,486,931 user ratings"
#> [5] "8.9 based on 749,563 user ratings"

ratings |>
  mutate(
    rating_n = html |> html_elements("td strong") |> html_attr("title")
  ) |>
  separate_wider_regex(
    rating_n,
    patterns = c(
      "[0-9.]+ based on ",
      number = "[0-9,]+",
      " user ratings"
    )
  ) |>
  mutate(
    number = parse_number(number)
  )

24.7 動的なサイト

これまでは、html_elements() がブラウザで見えるものを返し、それをどうパースし、整然とした（tidy）データフレームにどうまとめるかに焦点を当ててきました 。しかし時折、html_elements() たちがブラウザで見えるものとは全く異なるものを返すサイトに突き当たることがあります 。多くの場合、それは JavaScript でページのコンテンツを動的に生成しているウェブサイトをスクレイピングしようとしているためです 。rvest は生の HTML をダウンロードし、JavaScript を実行しないため、現在のところこれらには対応していません 。このようなサイトをスクレイピングすることも可能ですが、rvest はよりコストのかかるプロセス、つまりすべての JavaScript の実行を含むウェブブラウザの完全なシミュレーションを使用する必要があります 。この機能は執筆時点では利用可能ではありませんが、現在積極的に取り組んでおり、あなたがこれを読む頃には利用可能になっているかもしれません 。これは、実際に Chrome ブラウザをバックグラウンドで実行する chromote パッケージ を使用し、人間がテキストを入力したりボタンをクリックしたりするように、サイトと対話するための追加ツールを提供します 。詳細は rvest のウェブサイト を確認してください。

24.8 まとめ

この章では、ウェブページからデータをスクレイピングする理由、すべきでない理由、そしてその方法について学びました 。まず HTML の基本と、特定の要素を参照するための CSS セレクタの使用について学び、次に rvest パッケージを使って HTML から R へデータを取り出す方法を学びました。その後、2 つのケーススタディでウェブスクレイピングの実演を行いました。rvest パッケージのウェブサイトからスター・ウォーズ映画のデータをスクレイピングする単純なシナリオと、IMDb からトップ 250 の映画をスクレイピングするより複雑なシナリオです 。ウェブからデータをスクレイピングする技術的な詳細は複雑になることがありますが、法的および倫理的な考慮事項はさらに複雑になる可能性があります 。データスクレイピングを始める前に、これら両方について自分自身で学ぶことが重要です 。

これで、本書のインポート（import）部分が終わりました。ここでは、データがある場所（スプレッドシート、データベース、JSON ファイル、ウェブサイト）から、R の整然とした形式へとデータを取り込むテクニックを学びました。次は、プログラミング言語としての R を最大限に活用するという新しいトピックに目を向ける時間です。
