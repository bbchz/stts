8  ワークフロー：助けを求める

この本は孤島ではありません 。Rをマスターするための唯一無二のリソースというものは存在しません 。本書で説明したテクニックを自分のデータに適用し始めると、すぐに私たちが答えていない疑問にぶつかるでしょう 。このセクションでは、助けを得るためのいくつかのヒントと、学習を続けるための方法について説明します 。

【Geminiの補足】: 「No man is an island（人は一人では生きていけない／人は孤島ではない）」という、詩人ジョン・ダンの有名な一節を引用した比喩です。ここでは、学習において一冊の本だけに頼るのではなく、周囲のリソースやコミュニティを活用することの重要性を説いています。

8.1 Googleはあなたの味方です

行き詰まったら、まずはGoogleから始めましょう 。クエリに「R」を加えるだけで、関連する結果に絞り込むのに十分なことがよくあります。検索が役に立たない場合は、R固有の結果が利用可能でないことを意味することが多いです 。さらに、「tidyverse」や「ggplot2」といったパッケージ名を追加すると、自分にとってより馴染みのあるコードに結果を絞り込むのにも役立ちます（例：「Rで箱ひげ図を作る方法」 vs 「Rでggplot2を使って箱ひげ図を作る方法」） 。

Googleは特にエラーメッセージに役立ちます 。エラーメッセージが表示されて意味が全くわからない場合は、ググってみてください！ 。過去に誰かが同じように困っていた可能性が高く、ウェブ上のどこかに助けがあるはずです 。（エラーメッセージが英語でない場合は、Sys.setenv(LANGUAGE = "en") を実行してからコードを再実行してください。英語のエラーメッセージの方が助けが見つかる可能性が高くなります 。）

Googleで解決しない場合は、Stack Overflow を試してみてください 。まずは [R] を含めて既存の回答を探すことに少し時間をかけ、検索範囲をRを使用する質問と回答に限定しましょう 。

8.2 reprexを作成する

Google検索で有益な情報が見つからない場合は、reprex（minimal reproducible exampleの略、最小限の再現可能な例）を用意するのが非常に良いアイデアです 。優れたreprexがあれば他の人があなたを助けやすくなりますし、それを作成する過程で自分で問題に気づくこともよくあります 。reprexの作成には2つの要素があります ：

  • 第一に、コードを再現可能（reproducible）にする必要があります 。これは、すべてを網羅する必要があることを意味します。つまり、library() の呼び出しを含め、必要なオブジェクトをすべて作成します 。これを確実に行う最も簡単な方法は、reprexパッケージを使用することです 。
  • 第二に、それを最小限（minimal）にする必要があります 。問題に直接関係のないものはすべて取り除いてください 。これには通常、実際に直面しているものよりもはるかに小さく単純なRオブジェクトを作成するか、組み込みデータを使用することが含まれます 。

大変な作業に聞こえるかもしれませんが、それには大きな見返りがあります：

  • 80%の場合、優れたreprexを作成することで問題の原因が明らかになります 。自己完結した最小限の例を書き上げるプロセスによって、自分自身の疑問に答えられるようになることが驚くほどよくあります 。
  • 残りの20%の場合、他の人が簡単に試せる形で問題の本質を捉えることができます 。これにより、助けを得られる可能性が大幅に高まります！ 。

手作業でreprexを作成すると、うっかり何かを忘れてしまい、他の人のコンピュータでコードが実行できなくなることがよくあります 。tidyverseの一部としてインストールされるreprexパッケージを使用して、この問題を回避しましょう 。

例えば、このコードをクリップボードにコピーしたとします（RStudio ServerやCloudの場合は選択します）：

y <- 1:4
mean(y)

そして reprex() を呼び出します。デフォルトの出力はGitHub用にフォーマットされます ：

reprex::reprex()

RStudioのViewer（RStudioを使用している場合）またはデフォルトのブラウザに、綺麗にレンダリングされたHTMLプレビューが表示されます 。reprexは自動的にクリップボードにコピーされます（RStudio ServerやCloudでは、自分でコピーする必要があります） ：

``` r
y <- 1:4
mean(y)
#> [1] 2.5
```

このテキストはMarkdownと呼ばれる特別な方法でフォーマットされており、Stack OverflowやGitHubなどのサイトに貼り付けると、自動的にコードのようにレンダリングされます 。GitHubでレンダリングされたMarkdownは次のようになります ：

y <- 1:4
mean(y)
#> [1] 2.5

他の誰でもこれをすぐにコピー、貼り付け、実行できます 。例を再現可能にするために含める必要があるのは、必要なパッケージ、データ、コードの3つです ：

  1. パッケージはスクリプトの先頭でロードし、例にどれが必要か分かりやすくする必要があります 。これは各パッケージの最新バージョンを使用しているか確認する良い機会でもあります 。パッケージをインストールまたは最後に更新してから、修正されたバグが見つかるかもしれません 。tidyverseのパッケージの場合、最も簡単な確認方法は tidyverse_update() を実行することです 。
  2. データを含める最も簡単な方法は、dput() を使用してデータを再作成するために必要なRコードを生成することです 。例えば、Rで mtcars データセットを再作成するには、以下の手順を実行します：
    1. Rで dput(mtcars) を実行する
    2. 出力をコピーする
    3. reprexの中で mtcars <- と入力し、貼り付ける 。 問題が明らかになる範囲で、データの最小限のサブセットを使用するようにしてください 。
  3. コードが他の人にとって読みやすくなるように少し時間をかけましょう：
    • スペースを使用し、変数名が簡潔かつ情報量豊かなものであることを確認してください 。
    • コメントを使用して、どこに問題があるかを示してください 。
    • 問題に関係のないものはすべて取り除くよう最善を尽くしてください 。コードが短ければ短いほど理解しやすくなり、修正も容易になります 。

最後に、新しいRセッションを開始してスクリプトをコピー＆ペーストし、実際に再現可能な例が作成できたかを確認して終了します 。

reprexの作成は些細なことではなく、真に最小限の優れたreprexを作成できるようになるには練習が必要です 。しかし、コードを含めた質問をすることを学び、再現可能にするために時間を投資することは、Rを学びマスターするにつれて報われ続けるでしょう 。

8.3 自分自身への投資

問題が発生する前に、自分自身で解決できるよう準備しておく時間も必要です 。毎日少しずつRの学習に時間を投資することは、長期的には大きな見返りとなります 。一つの方法は、tidyverseチームの活動を tidyverseブログ でフォローすることです 。より広くRコミュニティの動向を把握するには、R Weekly を読むことをお勧めします。これは、毎週Rコミュニティで最も興味深いニュースを集約するコミュニティの取り組みです 。

8.4 まとめ

この章で、本書の「一連の流れ（Whole Game）」の部分は終了です 。これで、データサイエンスのプロセスの最も重要な部分である可視化、変形、整然化、インポートを一通り見たことになります。プロセス全体の全体像を把握できたので、ここからは小さなピースの詳細に入っていきます。

本書の次のパート「可視化（Visualize）」では、グラフの文法とggplot2によるデータ可視化を深く掘り下げ、これまでに学んだツールを使って探索的データ分析を行う方法を紹介し、コミュニケーションのためのプロット作成の優れた慣行を導入します。
