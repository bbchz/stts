はじめに

データサイエンスは、生のデータを理解、洞察、そして知識へと変えることができるエキサイティングな分野です 。『R for Data Science』の目的は、データサイエンスを効率的かつ再現可能な形で実行するためのRの最も重要なツールを学び、その過程を楽しみながら身につけていただくことです 😃 。本書を読み終える頃には、Rの優れた機能を活用して、多種多様なデータサイエンスの課題に取り組むためのツールが手元に揃っているはずです 。

学べること

データサイエンスは広大な分野であり、一冊の本ですべてをマスターすることは不可能です 。本書の狙いは、最も重要なツールの強固な基礎を築き、必要に応じてさらに深く学ぶためのリソースを見つけられるだけの知識を提供することにあります 。典型的なデータサイエンスプロジェクトのステップをモデル化したものが Figure 1 です 。

まず、データをRに読み込む（import）必要があります。これは通常、ファイル、データベース、またはウェブアプリケーションプログラミングインターフェース（API）に保存されているデータを取り込み、Rのデータフレームにロードすることを意味します 。データをRに取り込めなければ、データサイエンスを始めることはできません 。

データを読み込んだら、それを整然化（tidy）するのが良い方法です 。データの整然化とは、データセットの持つ意味（セマンティクス）と保存形式が一致するように、一貫した形式で保存することを指します 。簡単に言えば、データが整然としているとき、各列は変数であり、各行は観測値となります 。構造が一貫していることで、異なる関数に適合させるためにデータ形式と格闘するのではなく、データに関する疑問に答えることに集中できるようになるため、整然としたデータは重要です 。

データが整然としたら、次のステップは通常、それを変形（transform）することです 。変形には、特定の都市の人々や昨年のデータだけを抽出するといった関心のある観測値の絞り込み、距離と時間から速度を計算するといった既存の変数から新しい変数を作成すること、そして件数や平均値などの要約統計量を計算することが含まれます 。整然化と変形は合わせてラングリング（wrangling）と呼ばれます。データを扱いやすい形にする作業は、しばしば格闘（wrangle）のように感じられるからです 。

必要な変数が揃い、データが整然とした形になれば、知識を生み出すための2つの主要なエンジンである「可視化」と「モデリング」の出番です 。これらには相補的な長所と短所があるため、実際のデータ分析では両者の間を何度も行き来することになります 。

可視化（Visualization）は、本質的に人間中心の活動です。優れた可視化は、予想もしなかったことを示したり、データに対する新たな疑問を投げかけたりします 。また、問いが間違っていることや、別のデータを収集する必要があることを示唆してくれることもあります 。可視化は驚きを与えてくれますが、人間による解釈を必要とするため、大規模なスケーリングにはあまり向いていません 。

モデル（Models）は可視化を補完するツールです。問いが十分に精密になれば、モデルを使ってそれに答えることができます 。モデルは本質的に数学的または計算的なツールであるため、一般的にスケーリングに優れています 。スケーリングが難しい場合でも、通常は人間の脳を増やすよりもコンピューターを買い足す方が安上がりです 。しかし、すべてのモデルには仮定があり、その性質上、モデル自身が自分の仮定に疑問を持つことはできません 。つまり、モデルが根本的な驚きを与えてくれることはありません 。

データサイエンスの最後のステップは、あらゆるデータ分析プロジェクトにおいて極めて重要な共有（communication）です 。モデルや可視化によってどれほど深くデータを理解できたとしても、その結果を他者に伝えることができなければ意味がありません 。

これらすべてのツールを支えているのがプログラミング（programming）です。プログラミングは、データサイエンスプロジェクトのほぼすべてのパートで使用する横断的なツールです 。優れたデータサイエンティストになるためにプログラミングの達人である必要はありませんが、プログラミングを学ぶことは報われます。より良いプログラマーになることで、共通のタスクを自動化し、新しい問題をより簡単に解決できるようになるからです 。

これらのツールはあらゆるプロジェクトで使用しますが、それだけでは不十分な場合もあります 。ここには「80対20の法則」が当てはまります。本書で学ぶツールを使えば、あらゆるプロジェクトの約80%に対処できますが、残りの20%に対処するには他のツールが必要になります 。本書を通じて、さらに詳しく学べるリソースを随時紹介していきます 。

本書の構成

前述したデータサイエンスツールの説明は、分析でそれらを使用する順序にほぼ沿っています（もちろん、実際には何度もサイクルを繰り返します） 。しかし、私たちの経験上、データの読み込みと整然化を最初に学ぶのはあまり効率的ではありません。なぜなら、それらの作業は80%の時間は単調で退屈であり、残りの20%の時間は奇妙でフラストレーションが溜まるものだからです 。新しい科目を学び始める場所としては適切ではありません 。

代わりに本書では、すでに読み込まれ整然化されたデータの可視化と変形から始めます 。そうすることで、自分自身のデータを取り込み、整然化する段階になったとき、その苦労に値する結果が待っていることを知っているため、モチベーションを高く保つことができるからです 。各章では、「まず意欲を高める例から始めて全体像を示し、その後に細部を深く掘り下げる」という一貫したパターンに従うようにしています 。本書の各セクションには、学んだことを練習するための演習問題が用意されています 。演習をスキップしたくなるかもしれませんが、実際の問題で練習することに勝る学習方法はありません 。

本書で扱わないこと

本書でカバーしていない重要なトピックがいくつかあります 。できるだけ早く分析を始められるよう、エッセンスに冷徹なまでに焦点を絞ることが重要だと考えているからです 。そのため、すべての重要なトピックを網羅することはできません。

モデリング

モデリングはデータサイエンスにとって非常に重要ですが、巨大なトピックであり、残念ながら本書で十分な解説を行うスペースがありません 。モデリングについてさらに詳しく学びたい場合は、私たちの同僚であるMax KuhnとJulia Silgeによる『Tidy Modeling with R』を強くお勧めします 。この本では、本書で使用するtidyverseパッケージと多くの慣習を共有している tidymodels ファミリーのパッケージについて学ぶことができます 。

ビッグデータ

本書は、メモリに収まる程度の小規模なデータセットを主な対象としています 。小規模なデータでの経験なしにビッグデータに取り組むことはできないため、ここから始めるのが適切です 。本書で学ぶツールの大部分は、数百メガバイトのデータを容易に扱うことができ、少し注意を払えば数ギガバイトのデータも処理できます 。また、ビッグデータの保存によく使われるデータベースやParquetファイルからデータを取り出す方法も紹介します 。データセット全体を操作する必要はありません。関心のある問いに答えるためには、そのサブセットやサブサンプルがあれば十分だからです。

もし日常的にさらに大規模なデータ（10〜100GBなど）を扱っているなら、data.table について学ぶことをお勧めします 。本書でこれを教えないのは、tidyverseとは異なるインターフェースを持ち、異なる慣習を学ぶ必要があるためです 。しかし、非常に高速であり、大規模なデータを扱う場合には、学習に時間を投資するだけの価値があるパフォーマンス上のメリットがあります 。

Python、Julia、その他の言語

本書では、Python、Julia、またはデータサイエンスに有用なその他のプログラミング言語については一切学びません 。これはそれらのツールが劣っていると考えているからではありません。決してそんなことはありません！ 実際、ほとんどのデータサイエンスチームは複数の言語を組み合わせて使っており、多くの場合、少なくともRとPythonの両方を使用しています 。しかし、一度に一つのツールをマスターするのが最善であり、Rはそのスタート地点として最適だと私たちは強く信じています 。

前提条件

本書を最大限に活用していただくために、読者の皆様についていくつかの仮定を置いています 。一般的な数値リテラシーがあり、基本的なプログラミング経験があれば助けになります 。プログラミングが全く初めての方は、Garrettによる『Hands on Programming with R』が本書の貴重な副読本になるでしょう 。本書のコードを実行するには、R、RStudio、そして tidyverse と呼ばれるRパッケージのコレクション、およびその他のいくつかのパッケージの4つが必要です 。パッケージは、再現可能なRコードの基本単位です。再利用可能な関数、その使用方法を説明するドキュメント、およびサンプルデータが含まれています 。

R

Rをダウンロードするには、CRAN（Comprehensive R Archive Network）のサイト https://cloud.r-project.org にアクセスしてください 。Rの新しいメジャーバージョンは年に一度公開され、マイナーリリースは年に2〜3回あります 。定期的にアップデートすることをお勧めします。アップグレード（特にメジャーバージョンの場合、すべてのパッケージの再インストールが必要になるなど）は少し手間がかかることもありますが、先延ばしにすればするほど状況は悪化します 。本書では R 4.2.0 以降を推奨します。

RStudio

RStudioはRプログラミングのための統合開発環境（IDE）であり、https://posit.co/download/rstudio-desktop/ からダウンロードできます 。RStudioは年に数回アップデートされ、新しいバージョンが出ると自動的に通知されるため、こまめにチェックする必要はありません 。最新の優れた機能を利用するために、定期的にアップグレードすることをお勧めします 。本書では、少なくとも RStudio 2022.02.0 以降を使用していることを確認してください 。

RStudioを起動すると（Figure 2）、インターフェースに2つの主要な領域が表示されます。コンソール（Console）ペインと出力（Output）ペインです 。今のところ知っておくべきことは、コンソールペインにRコードを入力してEnterキーを押すと実行されるということだけです 。進めるにつれて、より詳しく学んでいきましょう！

RStudioのすべての機能についての包括的な概要を知りたい場合は、RStudioユーザーガイド（https://docs.posit.co/ide/user）を参照してください 。

tidyverse

また、いくつかのRパッケージをインストールする必要があります 。Rパッケージとは、base Rの機能を拡張する関数、データ、ドキュメントのコレクションです 。パッケージを使いこなすことが、Rを使いこなす鍵となります。本書で学ぶパッケージの大部分は、いわゆる tidyverse（タイディバース）の一部です 。tidyverseに含まれるすべてのパッケージは、データとRプログラミングに関する共通の哲学を共有しており、連携して動作するように設計されています 。

以下の1行のコードで、tidyverse全体をインストールできます。

install.packages("tidyverse")

お使いのコンピューターのコンソールにこのコードを入力し、Enterキーを押して実行してください 。RがCRANからパッケージをダウンロードし、コンピューターにインストールします 。パッケージ内の関数やオブジェクト、ヘルプファイルは、library() でロードするまで使用できません 。インストールが完了したら、library() 関数を使ってロードできます。

library(tidyverse)

#> ✔ forcats   1.0.1     ✔ stringr   1.5.2
#> ✔ ggplot2   4.0.0     ✔ tibble    3.3.0
#> ✔ lubridate 1.9.4     ✔ tidyr     1.3.1
#> ✔ purrr     1.1.0     
#> ── Conflicts ─────────────────────────────────────── tidyverse_conflicts() ──
#> ✖ dplyr::filter() masks stats::filter()

これにより、tidyverseが dplyr, forcats, ggplot2, lubridate, purrr, readr, stringr, tibble, tidyr という9つのパッケージをロードしたことがわかります 。これらはほぼすべての分析で使用するため、tidyverseのコア（core）パッケージと見なされています 。tidyverseのパッケージは頻繁に変更されます。tidyverse_update() を実行することで、アップデートが利用可能かどうかを確認できます 。

その他のパッケージ

tidyverse以外にも、別のドメインの問題を解決するため、あるいは異なる設計原則に基づいて作られた優れたパッケージが数多く存在します 。それらは単に「異なる」だけであり、劣っているわけではありません 。言い換えれば、tidyverseの補集合は「メッシーバース（しっちゃかめちゃかな世界）」ではなく、相互に関連する他の多くのパッケージ群なのです 。

【Geminiの補足】: 「messyverse」は、tidyverse（整然とした世界）の対義語として著者が作った造語です。日本語で言うところの「カオスな世界」といったニュアンスですが、Rの世界にはtidyverse以外にも洗練されたパッケージ群が多数存在することを、ユーモアを交えて表現しています。

Rを使ったデータサイエンスプロジェクトに取り組むにつれ、新しいパッケージやデータに関する新しい考え方を学んでいくことになります 。本書でも、tidyverse以外のパッケージを多く使用します 。例えば、Rを学ぶ過程で興味深いデータセットを提供してくれる以下のパッケージなどです。

install.packages(
  c("arrow", "babynames", "curl", "duckdb", "gapminder",
    "ggrepel", "ggridges", "ggthemes", "hexbin", "janitor", "Lahman",
    "leaflet", "maps", "nycflights13", "openxlsx", "palmerpenguins",
    "repurrrsive", "tidymodels", "writexl")
  )

また、一回限りの例のために他のパッケージもいくつか使用します 。今すぐインストールする必要はありませんが、次のようなエラーが表示された場合は、

library(ggrepel)
#> Error in library(ggrepel) : there is no package called ‘ggrepel’

install.packages("ggrepel") を実行してパッケージをインストールする必要があることを覚えておいてください 。

Rコードの実行

前のセクションでは、Rコードを実行する例をいくつか示しました 。本書のコードは次のように表示されます。

1 + 2
#> [1] 3

これをご自身のコンソールで実行すると、次のようになります。

> 1 + 2
[1] 3

主な違いが2つあります 。コンソールでは、プロンプトと呼ばれる > の後に入力しますが、本書ではプロンプトを表示していません 。本書では、出力結果の前に #> を付けてコメントアウトしていますが、コンソールではコードの直後に表示されます 。電子書籍版をお読みであれば、これらの違いを気にすることなく、コードを簡単にコピーしてコンソールに貼り付けることができます 。

本書では、コードを参照するために一貫した慣習を使用しています。

  • 関数はコードフォントで表示され、sum() や mean() のように後に括弧が続きます 。
  • その他のRオブジェクト（データや関数の引数など）は、flights や x のように括弧なしのコードフォントで表示されます 。
  • どのパッケージのオブジェクトであるかを明確にするために、dplyr::mutate() や nycflights13::flights のように、パッケージ名の後に2つのコロン（::）を付けて表記することもあります 。これはRの正しいコード形式でもあります。
